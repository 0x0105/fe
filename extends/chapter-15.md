前端工程师写给后台看的 API 设计指南
===

设计原则 
 
 - 直观。一看就知道是干什么的，比如 getElementById。
 - 语义化。
 - 使用标准。Restful
 - 容易记住。避免缩写
 - 保持一致。

考虑因素：

 - 工作量
 - 复杂度
 - 人力。

交付软件就好比是上战场，当一端的人力不够，而另外一端人力充足时，就应该适当作一些妥协。

交给后台完成的情形：

 - 前端、应用端使用同一套 API，那么当出现 API 前后端皆可时，应该交给后台。这是从工作量的计算上来看的，后台实现这个功能一天，移动端、前端各一天，那么就应该交由后台去做。Time is money.
